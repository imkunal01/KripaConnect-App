# Frontend-Backend Integration Guide

## Overview
Connect existing frontend with backend functionality. Replace all dummy data and hardcoded features with working backend integrations.

---
## 1. Navigation Bar

### Requirements:
- **Categories Button**: Redirect to category listing page
- **B2B Section**: Navigate to retailer portal with bulk pricing
- **Favorites**: User can save items for later purchase
- **Cart**: Add items and proceed to checkout/payment
- **Profile**: View user profile and account data

## 2. Home Page
### Hero Section:
- Implement working search bar for product search
- Connect all CTA buttons to respective pages

### Categories Section:
- Display category cards from backend
- Clicking category → filter products by that category

### Trending Products:
- Replace dummy data with real backend products
- Display dynamic product listings

### Footer:
- Make all footer links functional

---

## 3. Hamburger Menu

### Navigation Items:
- **Home**: Redirect to homepage
- **Product Listing**: Navigate to products page

---

## 4. Product Listing Page

### Left Sidebar Filters:
- Category filter
- Price range slider
- Brand filter
- Availability filter (in stock/out of stock)

### Top Section:
- Search bar
- Sort dropdown (price: low to high, high to low, newest, etc.)

### Product Grid:
- Display all products from backend
- Add to cart button
- Add to favorites button

### Actions:
- Add items to cart
- Mark products as favorite

## 5. Product Detail Page

### Display Information:
- Product name
- Product image
- Quantity selector
- Price
- Add to cart button
- Add to favorites button

**Note**: Keep it minimal - only essential product information

## 6. Authentication Pages

### Login/Signup:
- Connect with backend authentication
- Implement session management (persist login on refresh)
- Add frontend validation:
  - Email format validation
  - Password strength requirements
- Keep existing design - only add backend integration

## 7. Shopping Cart & Checkout

### Shopping Cart Page:
- Keep current design
- List added products
- Display order summary
- Show discount (if applicable)
- Show total amount
- Checkout button

**Remove**: Coupon section below product listing

### Checkout Page:
- Integrate Razorpay payment gateway
- Payment options: **UPI and COD only** (no credit card)
- Allow address editing (back button)
- Allow payment method changes

### Order Confirmation:
- Show Flipkart-style order placed animation
- Confirm order placement

### Sections:

#### Order History:

#### Profile Settings:

#### Favorites:

#### Logout:

## 9. Retailer Portal (B2B Section)

### Access Control:
- Only visible to users with "retailer" role
- Hidden from regular customers

### Features:
- Display products with bulk pricing
- Show wholesale rates from product schema
- Similar layout to customer product listing
- Bulk order functionality

---

## 10. Admin Dashboard ⚠️ CRITICAL

### Dashboard Overview:
Display key metrics:
- **Total Revenue**: Sum of all completed orders
- **Total Users**: Count separately (customers vs retailers)
- **Total Orders**: Count of all orders
- **Revenue Chart**: Monthly revenue analysis (line/bar chart)
- **Stock Distribution**: Pie chart showing inventory levels

### Product Management:
- Upload product images via Cloudinary
- Reference backend product schema for fields
- CRUD operations on products:
  - Add new products
  - Edit existing products
  - Delete products
  - Update stock levels
  - Set regular & bulk pricing

### Category Management:
- Add new categories
- Delete existing categories
- **Auto-update**: Changes reflect on customer & retailer pages immediately

### Order Tracking:
- View all user orders
- Update order status:
  - Order Placed
  - Processing
  - Shipped
  - Out for Delivery
  - Delivered
- Send status updates to users

### User Management:
- View all users (customers & retailers)
- CRUD operations:
  - Create new users
  - Edit user details
  - Delete users
  - Change user roles

### Inventory Management:
- List all products
- Update stock quantities
- Mark products as out of stock
- Edit bulk pricing
- Edit regular pricing
- Low stock alerts

### Payment Management:
- Track payment transactions
- View payment methods used
- Refund management
- Payment status tracking
- Revenue reports

### Logout:
- Secure admin logout
- Clear admin session

---

## Technical Requirements

### Backend Integration:
- Use existing backend API endpoints
- Implement proper error handling
- Add loading states for async operations

### State Management:
- Maintain cart state
- Persist favorites
- Handle user sessions

### Image Upload:
- Cloudinary integration for product images
- Profile photo uploads

### Database Schema Checks:
- Verify product schema includes:
  - Regular price
  - Bulk price
  - Stock quantity
  - Category
  - Images (Cloudinary URLs)
- Verify user schema includes:
  - Role (customer/retailer/admin)
  - Profile photo
  - Favorites array
  - Order history

### Security:
- Role-based access control
- Protected admin routes
- Secure payment processing
- Input validation (frontend & backend)
